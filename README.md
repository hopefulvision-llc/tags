# Tags

> **Layer:** Ontology  
> **Status:** Working Canon  
> **Audience:** Researchers, builders, editors, knowledge architects  
> **Related:** `nous-object-id`, `nousos`, `technomysticism`, `.github`

## What this repo is
Tags is the semantic and ontological index layer of the HopefulVision ecosystem. It defines recurring concepts, themes, categories, and cross-references that help the broader body of work remain navigable, interoperable, and meaningfully connected across repositories.

This repo is the vocabulary backbone of the ecosystem.

## Why it exists
As a knowledge ecosystem grows, concepts begin to drift, overlap, and fragment unless a shared semantic structure exists. Tags exists to stabilize language, support discoverability, and make it easier to trace how ideas relate across philosophy, governance, architecture, and prototypes.

It helps the ecosystem think in a connected way instead of as isolated documents.

## What Tags is not
Tags is **not**:
- the primary identity and embodiment framework, which belongs to `nous-object-id`
- the core systems architecture layer, which belongs to `nousos`
- just a random collection of labels without structural intent

Tags is the **ontology and semantic navigation layer**.

## How it fits the ecosystem
This repo connects most directly to:
- `nous-object-id`, which defines structured identity and classification logic
- `nousos`, which depends on consistent concepts across systems
- `technomysticism`, which contributes key worldview vocabulary
- `.github`, which may point people to this shared semantic layer across the org

## Implementation Notes
**Core Design Principles**:

**1. Understanding Over Agreement**
- Success metric: Can you accurately state opponent's position?
- Require paraphrasing before counter-arguing
- Reward clarity, not rhetorical victory
- Neutral third parties verify understanding

**2. Structure Over Chaos**
- Claim → Evidence → Reasoning format required
- No ad hominem allowed by design
- Sources must be cited and verifiable
- Arguments build on each other, not scatter

**3. Coherence-Gated Participation**
- Must be 75%+ coherent to post (HRV measurement)
- Reactive rants blocked automatically
- System suggests "breathe first" when agitated
- Conscious discourse only

**4. Truth Weighting**
- Not popularity voting
- Verified facts weighted higher
- Expert consensus acknowledged
- Uncertainty clearly labeled

**5. Good Faith Assumption**
- Platform design assumes people want truth
- Trolls naturally filtered by coherence requirement
- Bad actors can't sustain high coherence while trolling

**Key Features**:

**Structured Dialogue Trees**:
```
Claim: "X policy would reduce poverty"
├─ Evidence: [Studies, data, sources]
├─ Reasoning: [Logical connection]
├─ Counter-claim: "X policy has unintended consequences"
│  ├─ Evidence: [Alternative data]
│  ├─ Reasoning: [Different interpretation]
│  └─ Synthesis: [Areas of agreement and disagreement]
└─ Truth Assessment: [Community evaluation with reasoning]
```

**Understanding Verification**:
Before responding, must demonstrate understanding:
- Paraphrase opponent's position
- Opponent confirms accuracy
- Only then can counter-argue
- Prevents strawman arguments automatically

**Source Verification**:
- All claims require citations
- System checks source credibility
- Peer review process for contested sources
- Conflicting studies shown side-by-side

**Cognitive Bias Detection**:
- AI identifies common fallacies
- Suggests reframing for clarity
- Shows how argument might be misinterpreted
- Encourages steel-manning over straw-manning

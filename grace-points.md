## Implementation Notes

### Point Generation Categories

**1. Coherence Points (40% weight)**
- HRV coherence >75%: 1 point per minute
- HRV coherence 50-75%: 0.5 points per minute
- Meditation sessions: Bonus multiplier
- Consistency streaks: Additional bonuses

**2. Community Healing (30% weight)**
- Verified teaching/mentoring: 10 points per hour
- Community support work: 5 points per hour
- Conflict resolution facilitation: 15 points per session
- Emotional healing work: Variable based on verification

**3. Planetary Action (20% weight)**
- Regenerative activities: Points based on impact
- Activism/organizing: Community-verified points
- Ecosystem restoration: Measured contributions
- Climate healing work: Verified activities

**4. Consciousness Practice (10% weight)**
- Documented meditation: 2 points per 30min
- Ceremony participation: Variable points
- Inner work documentation: Community recognition
- Teaching/sharing: Knowledge contribution points

### Anti-Gaming Mechanisms
- **HRV Spoofing Detection:** Algorithmic patterns identify fake coherence
- **Community Verification:** Healing work requires witness confirmation
- **Random Audits:** Periodic deep verification of claimed activities
- **Reputation Staking:** Verifiers stake reputation on accuracy
- **Pattern Analysis:** AI detection of suspicious point generation
- **Cooldown Periods:** Prevent point farming through artificial repetition

### Privacy Protections
- HRV data stays local, only coherence scores transmitted
- Community verification is consent-based, not mandatory
- Pseudonymous point tracking available
- Right to disconnect from tracking without penalty
- Data deletion/opt-out always available

## Technical Architecture (Draft)

### Data Structure
```
GracePointEntry {
  userId: string (pseudonymous)
  timestamp: datetime
  category: enum [coherence, healing, planetary, practice]
  points: number
  verification: {
    method: enum [hrv, community, self-report, algorithmic]
    verifier: string (if community verified)
    confidence: number (0-1)
  }
  metadata: {
    duration: number (for time-based activities)
    witnesses: string[] (community verification)
    notes: string (optional context)
  }
}
```

### Point Calculation Algorithm (Simplified)
```
dailyPoints = 
  (coherenceMinutes * coherenceWeight * 1) +
  (healingHours * healingWeight * 10) +
  (planetaryActions * planetaryWeight * variablePoints) +
  (practiceMinutes * practiceWeight * 0.067)
  
weeklyBonus = consistencyStreak * 5
monthlyTotal = sum(dailyPoints) + weeklyBonus
```

### Verification Workflow
1. User performs activity (meditation, healing work, etc.)
2. Activity logged (auto via HRV, or manual entry)
3. Points provisionally awarded
4. Verification process (if required):
   - Community verifiers review claim
   - AI pattern analysis checks for gaming
   - Reputation-weighted consensus
5. Points confirmed or adjusted
6. Ledger updated transparently

## Known Risks
- **Goodhart's Law:** When measure becomes target, it ceases to be good measure
- **Surveillance Creep:** Tracking expands beyond conscious consent
- **Inequality Reproduction:** Point-rich get richer through better access
- **Motivation Corruption:** External rewards corrupt intrinsic practice
- **Verification Burden:** Community verification becomes unpaid labor
- **Privacy Violations:** Data leaks expose consciousness practices

## Mitigations
- Regular community review of point algorithms
- Strict data minimization and privacy-by-design
- Baseline UBR floor regardless of points (safety net)
- Education about intrinsic vs extrinsic motivation
- Compensate verifiers with their own grace points
- Encryption, pseudonymity, and user control over data
